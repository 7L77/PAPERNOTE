
# 综述

## 背景
近年，随着人工智能技术的快速发展，神经网络模型在各个领域取得了显著的成果。

然而，神经网络模型的训练过程通常需要大量的计算资源和时间，这使得模型的部署和应用变得困难。

为了解决这一问题，研究人员提出了一种称为神经网络架构搜索（Neural Architecture Search，NAS）的方法。

然而，NAS 方法的搜索效率和搜索空间的大小仍然是一个挑战。

为此，研究人员提出了免训练代理（Zero-Cost Proxy，ZCP）的方法。
    ZCP 方法的基本思想是通过设计一个代理模型，来评估神经网络模型的性能，而无需实际训练模型。
    这种方法的优势在于，它可以在短时间内评估大量的架构，从而提高 NAS 方法的搜索效率。

## 已有的方法

### 免训练代理

#### AZ-NAS（CVPR 2024）
**核心思想**：要想在不训练网络的前提下更准确地判断架构优劣，必须从多个互补维度综合评估。

**四个可在一次前后向中高效计算的 proxy**：
- **sE（表达性）**：layer_features 特征上做协方差分解，计算主成分归一化后熵值。熵高意味着特征越各向同性，表达能力越强。
- **sP（渐进性）**：相邻 block 表达性差值的最小值。约束"深层特征空间应持续扩展"，不是只看某个局部峰值。
- **sT（可训练性）**：基于 Hutchinson 风格随机向量近似 block Jacobian，衡量梯度传播稳定性（谱范数接近 1 更好）。
- **sC（复杂度）**：直接使用 FLOPs 作为复杂度 proxy，在预算内偏好更高算力利用架构。

**非线性排名聚合**：`sum(log(rank/m))`，对低排名项惩罚更强，避免被高分项"冲掉"，鼓励均衡表现。

#### VKDNW (CVPR 2025)

**VKDNW**：基于 Fisher Information Matrix 的谱熵排序，用参数估计不确定性的"谱熵"来排序架构
提出问题：
1. 只看“网络在初始化时的性质”，就估计它将来是否容易训练、是否可能达到更高分类精度？
2. NAS 真正关心的是：**能不能把“最好的那些网络”排到前面**，而不是低性能网络之间排得多精确。

VKDNW思路：
	`vkdnw_dim`是参数张量个数
	`vkdnw_entropy`是Fisher Information Matrix 谱熵
        构造经验FIM = J^T J
        奇异值分解FIM，得到分位数
        归一化（奇异值/奇异值之和）+香农熵（-plogp）

引入 **nDCG** 作为更贴近实际 NAS 需求的评价指标

#### NEAR （ICLR 2025）
适用于：CNN
**NEAR**：利用各层预激活/后激活矩阵的 Effective Rank 估计网络质量，跨 NAS 搜索空间较稳健
提出问题：能否设计一个对激活函数更通用、跨空间更稳健、完全不需要标签的免训练代理？
研究动机：

思路：
    将effective rank 作为表达能力信号，设计免训练代理
    特征提取后，求奇异值，求香农熵，exp^香农熵=erank(A)
    将预激活矩阵和后激活矩阵的香农熵拼接起来，作为代理

#### SWAP （ICLR 2024）

**SWAP-NAS**：用"按神经元收集、按样本二值化"的激活模式基数衡量未训练网络可表达性
提出问题：
    现有激活模式度量本身的问题
        传统按“样本维”统计的 activation pattern 上界受 batch 大小限制，容易饱和
        一旦饱和，不同架构分数挤在一起，区分能力下降，甚至影响排序可靠性。
    现有 training-free 指标的问题
        对大模型有偏置，容易“分高但不一定更好”。
        与真实性能相关性不稳定，很多时候排不准。
        跨搜索空间/任务泛化差。

研究动机：
    提出了 sample-wise activation patterns（SWAP-Score）来扩大可区分空间；
    加 regularization 去缓解大模型偏置、实现可控模型大小搜索。

操作：
    SWAP 打分实际用的是 hook 到的中间 ReLU 特征,只取激活后的特征
    把激活模式从“按样本作为行”重排为“按神经元作为行”，放大可区分上界，提升排序分辨率。

#### NCD (ICCV 2025)

**NCD**：基于知识蒸馏的免训练架构评分

提出问题：像 NWOT/SWAP 这类 zero-cost proxy，在网络更深（卷积层更多）时，和真实性能的 Spearman 相关会掉到负数，导致“排得越靠前反而越差”。
	原因（作者结论）：高非线性累积会让 proxy 分数幅值失真，相关性翻转。

研究动机：发现并修复了 training-free NAS 里 activation-based proxy 的“负相关”问题。

操作：
1. SAM（Stochastic Activation Masking）：卷积里对激活做随机掩码，减轻非线性堆叠。
2. NIR（Non-linear Rescaling）：做非线性重标定（结合 BN/LN 机理）稳定评分。

#### MeCo(2023)

**MeCo**：基于最小特征值（Minimum Eigenvalue of Correlation）构造免训练代理，只需一次前向
适用：CNN

提出的问题：
1. 许多 zero-cost proxy 依赖梯度或多次前后向，评估开销仍然偏高，不利于大规模候选快速筛选。
2. 训练前排序需要一个更“轻量但有理论支撑”的信号，而不是纯启发式打分。

研究动机：
1. 能否仅用“一张随机样本 + 一次前向传播”就完成可用的架构排序。
2. 利用特征相关性的谱信息（相关矩阵最小特征值）作为可解释 proxy，并把它和收敛/泛化理论联系起来。

提出的操作：
1. 对每个候选架构只做一次前向，提取各层特征图。
2. 在每层上构建 Pearson 相关矩阵，计算最小特征值，并跨层求和得到 MeCo 分数。
3. 在搜索流程中用 MeCo 做候选构建与最终排序（Supplementary Algorithm 1 的两阶段流程）。
4. 针对“通道数变化大时相关性不稳”的问题，提出 MeCoopt：固定采样通道并按 `c(i)/n` 加权聚合，提升跨空间稳定性。


**WRCor**：统计不同样本在多层激活与梯度响应的相关系数矩阵，并对高层响应指数加权
适合需要接入随机搜索/RL/进化搜索的流程。

提出问题：
	大搜索空间里，proxy 与真实精度相关性会退化，甚至不稳定优于简单基线（如 Params/FLOPs）
	zero-shot proxy 常常只衡量单一属性（只看表达性或只看可训练性），在不同搜索空间下稳定性不足


研究动机：
核心假设是：**好架构应让不同样本的响应更“可分”**（跨样本相关性更低，log-det 更高）
高层特征对最终区分更关键，所以在响应相关矩阵上做了 **按层指数加权（2^l）**，形成 WRCor
用投票代理（如 SJW/SPW）进一步缓解“单一 proxy 不是全场景最优”的问题

操作：
forward 收集激活相关，backward 收集梯度相关
按层指数加权：$2^i$ 累加到 K，对应论文加权思想
对 K 做 slogdet，输出 WRCor 分数，并用于 random/RL/evolution 排序


**Dextr**：基于 SVD 特征条件数 + 外在曲率组合，同时覆盖收敛性/泛化性/表达性
CNN 与 ViT 搜索空间都可尝试（论文在 DARTS 和 AutoFormer 上都有实验）。

提出问题：
1. 大量方法只优化一侧：要么偏收敛/泛化，要么偏表达性，导致排序有偏。
2. 很多 zero-cost proxy 依赖标签，现实里标签常不可用。

研究动机：
	特征图共线性高会影响收敛与泛化；输出曲率能反映网络表达性。
	把这两类互补信息融合成一个分数（Dextr），让 proxy 更接近真实最终性能排序

Dextr 的本质是把两类信号融合：  
    收敛/泛化项 a=$log(1 + Σ_l 1/c_l(X))$，表达性项b = $log(1 + κ)$，最终 $Dextr = a*b/(a+b)$（Eq. 8）。
    收敛/泛化项：给网络所有模块注册 forward_hook，再前向 net(x)
    用各层特征的 σ_min/σ_max（即逆条件数）来近似$∑_l 1/c_l(X_ϕ)$ 
        每层取输出特征 fea，展平后做 svdvals，
        计算层分数$s_l=σ_min/σ_max$（逆条件数信号）
        汇总成 result_list。
    曲率（表达性）：调用 get_extrinsic_curvature_opt(...) 估计曲率 κ
        在 length.py 中构造轨迹输入 g(θ)，
        计算输出对 θ 的一阶导 v 和二阶导 a，
        累加曲率项（基于 v,a 的经典曲率表达式）得到 κ


衍生的问题：
	曲率估计需要高阶导，且 appendix 显示在 NATS-Bench-SSS 上原始 Dextr 会失稳（需 Dextropt 修正）。

#### W-PCA（ICLR 2025）
**W-PCA**：面向轻量语言模型搜索的无梯度代理，用 `W-PCA = #Params × V-PCA` 做训练前排序。

提出问题：
1. one-shot NAS 需要训练 supernet，LLM 搜索成本高。
2. 现有 zero-cost proxy 在 NLP 搜索空间上相关性不稳定，且不少方法依赖梯度，评估代价偏高。
3. 单一信号（只看参数量或只看某类统计量）容易有偏置。

研究动机：
1. 作者观察到 FFN 隐状态的 PCA 统计与下游性能趋势一致，初始化阶段也有预测力。
2. 希望构造一个“只前向、不反传”的代理，同时兼顾表征信息（PCA）和容量信息（参数规模）。

操作：
1. 在每层 FFN 上取中间表示 `H = XW1 + b1`，计算 PCA 维度阈值分数（V-PCA）。
2. 跨层聚合得到 `S(X)=sum_f S_f(X)`。
3. 用参数规模加权：`W-PCA(X)=w*S(X)`。
4. 用 W-PCA 作为适应度，配合遗传算法做架构搜索，再对最优结构进行预训练与微调（含 KD）。
