# 每日论文推荐

这是面向用户的一句话入口。对用户来说，正常只需要说一次：

- `今日论文推荐`
- `过去3天论文推荐`
- `过去一周论文推荐`
- `2026年论文推荐`
- `2026-01-01 到 2026-03-09 论文推荐`

## 执行原则

1. 先识别时间范围：
   - `今日论文推荐`、`每日推荐`、`今日论文` -> 当天
   - `过去3天论文推荐`、`最近3天论文` -> 3 天
   - `过去一周论文推荐`、`看看这周有啥论文` -> 7 天
   - `2026年论文推荐` -> `2026-01-01` 到 `2026-12-31`；如果今年尚未结束，则结束日期取今天
   - 显式日期范围（如 `2026-01-01到2026-03-09`）-> 解析为 `--start-date` 和 `--end-date`
2. 如果是相对时间窗口（今天 / 过去 N 天），运行：

```bash
python ../daily-papers/run_daily_recommendation.py --days N
```

默认当天可省略 `--days`。

3. 如果是显式日期范围 / 年度汇总，运行：

```bash
python ../daily-papers/run_daily_recommendation.py --start-date YYYY-MM-DD --end-date YYYY-MM-DD
```

4. 范围模式是“汇总推荐”，不是“今日流水线”：
   - 只生成汇总推荐文件
   - 不更新 `.history.json`
   - 不触发论文笔记批量生成
   - 不纳入 HF Trending（该源不支持历史日期）
5. 完成后，用一句话告诉用户：
   - 推荐文件已生成
   - 覆盖的准确日期范围
   - 如果是范围模式，明确说明这是汇总结果而非每日增量

## 重要约束

- 不要先要求用户手动跑 `跑一下论文抓取 / 点评 / 笔记`。
- 相对时间窗口可以继续视为完整流水线入口；显式日期范围则走“汇总推荐”模式，不要强行补笔记步骤。
- 如果用户明确只想跑其中一步，再交给对应 skill。

## 自动化

- 本 skill 本身就是“一步出结果”的入口。
- 如果用户想做本地定时任务，默认也应该触发这一句，而不是写死三条内部命令。
