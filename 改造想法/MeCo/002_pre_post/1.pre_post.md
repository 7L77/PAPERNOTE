### 思路2：特征值运算
按“每层 eigenvalues”理解，一般是：
- max：这一层最大特征值，max(lambda)
- mean：这一层特征值均值，mean(lambda)
- logdet：这一层矩阵的对数行列式，通常写成 sum(log(lambda_i + eps))
- erank：effective rank，有效秩  
    p_i = lambda_i^+ / sum(lambda^+)  
    erank = exp(-sum(p_i log p_i))
- stable_rank：稳定秩  
    如果是对“当前保存的相关矩阵”算，通常可写成  
    $$ stable\_rank = sum(lambda_i^2) / max(lambda)^2$$
下面这些情况才需要改代码重跑：

- 你要算的不是“相关矩阵 eigenvalues”的统计，而是“原始 activation 特征矩阵”的统计
- 你想改 sample_k，因为当前 eigenvalues 已经固定在当时采样的 8 个样本上了
- 你想换矩阵定义，比如从 corrcoef(xs) 改成 covariance、Gram matrix、或别的归一化方式

因为你这 300 个架构的 post 层数都固定为 8 层，mean = sum / 8，秩相关不会变。

模型就是按 layers=8 构建的
	model = Network(C=3, num_classes=10, layers=8, genotype=genotype).to(device)
post 特征提取是按 self.cells 一层一个地收
	modelV2.py 这里创建了 self.cells，数量就是 layers  
	modelV2.py这里的 extract_cell_features() 每个 cell append 一次 s1

post 返回的层数就是 len(self.cells) = layers = 8
pre-activation 不是 8 层，是因为它不是按 cell 取，而是对所有 nn.ReLU 注册 forward_pre_hook，每遇到一个 ReLU 就记一次
